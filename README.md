# 蚁坊 (Yifang)

蚁坊——蚂蚁工坊。蚂蚁个体简单，但通过分工和协作，建造出远超个体能力的复杂巢穴。

Yifang 不是又一个 AI 助手。它是一家 **AI 乙方公司**。

> **注意：** 本项目还处于早期探索阶段，是一次初步的尝试，不是一个完整可用的产品。架构和实现都在持续搭建和迭代中。

## 你是甲方

你提需求，Yifang 内部自己搞定分工、流程、质检。你不需要关心它内部怎么运转——就像你不需要关心乙方公司的日常管理。你只需要提需求，验收结果。

你面对的不是一个 Agent，而是一个**组织**。

## 为什么是组织，不是单个超级 Agent

当前 AI 应用的主流思路是：用一个更大、更强的模型，塞进更长的 prompt，期望它一次性给出完美答案。

这条路有天花板。

**三思而后行**——一个人想三遍，不如三个人各想一遍再碰。单个模型的能力是有限的，但如果让多个 Agent 从不同角度审视同一个问题、互相质疑、交叉验证，最终的输出质量可以超越任何单个模型。

这不是新发明。人类几千年前就想明白了这件事。

## 站在人类几千年的肩膀上

人类花了几千年时间回答一个问题：**怎么让一群人协作，产出超过个体之和的结果？**

答案沉淀成了螺旋模型、TDD、XP、Scrum、Holacracy、同意制……这些不是凭空发明的，是无数次失败后压缩出的集体智慧。

问题是，这些方法论在人类团队中总是打折执行——人会偷懒，会伤面子，会嫌流程麻烦。

**AI 是这些方法论的理想执行者。**

每个 Agent 能力一致，不存在水平参差；流程是代码，执行零成本；质疑不产生人际摩擦；角色切换即时，没有认知负担。人类发明的组织理论，终于遇到了能 100% 执行的团队。

Yifang 不发明新的组织理论，它把已经被人类验证过的方法论，用在 AI 上。

## 核心理念

**模型是灵智，架构是身体。**

我们不需要无限追求更强的模型。我们需要的是给普通的智能配上合适的认知结构——记忆、注意力、直觉、组织——让普通的智能，做出超越自身的事。

笨学生用对方法，也能解题。

## 当前实现

Yifang 当前的核心机制是**动态异步书面会议**：

- **秘书**管理信息流，压缩长内容，整理最终输出
- **主控**只做决策——召集谁、解散谁、继续还是收敛
- **安全审计**独立监控，不可被撤销，三级风险响应
- **质疑者**审视每一轮发言，找逻辑漏洞
- **专家团**按需召集，多角度并行分析，完成即离场

每轮都经过 质疑→审计→决策 的螺旋，最多 5 轮自动收敛。

详见 [设计哲学](docs/PHILOSOPHY.md) 和 [架构设计](docs/ARCHITECTURE.md)。

uv sync

uv run python main.py
```

    secretary.py     # 秘书

    protocol.py      # 数据结构
    prompts.py       # 提示词

docs/
  PHILOSOPHY.md      # 设计哲学
  ARCHITECTURE.md    # 架构设计
```
