# Yifang 设计哲学

---

## 一、核心信念

**模型是灵智，架构是身体。**

一缕灵智本身已经足够聪明。我们不需要无限追求更强的模型，我们需要的是给这缕灵智配上合适的认知结构——记忆、注意力、直觉、组织——让普通的智能，做出超越自身的事。

笨学生用对方法，也能解题。

---

## 二、组织观：人类几千年的智慧，AI 来执行

**人类最伟大的发明不是工具，是组织。**

从部落议事到现代公司，从军事参谋制到敏捷开发，人类花了几千年时间摸索出一个核心问题的答案：**怎么让一群智能体协作，产出超过个体之和的结果？**

这些答案——螺旋模型、TDD、XP、Scrum、Holacracy——不是凭空发明的，是无数次失败后沉淀下来的组织规律。它们是人类的"教训层"，是压缩过的集体智慧。

Yifang 的选择是：**不发明新的组织理论，而是把已经被验证的方法论，用在 AI 上。**

### AI 是这些方法论的理想执行者

人类发明的这些流程模型，在人类团队中总是打折执行。原因很简单：人是不稳定的。

| 人类团队的痛点 | AI 团队的天然优势 |
|---------------|------------------|
| 水平参差不齐，新手拖后腿 | 每个 Agent 都是同一缕灵智的实例，能力一致 |
| 流程太多，人会偷懒、抵触 | 流程是代码，执行零成本，不存在"嫌麻烦" |
| 结对审查伤感情 | Agent 没有自尊心，质疑不产生人际摩擦 |
| 角色切换有认知负担 | 不同 prompt 就是不同角色，切换即时 |
| 开会浪费时间，信息在嘴上丢失 | 书面异步会议，每个字都有记录 |
| Holacracy 在人类公司常常失败——太反直觉 | 对 AI 来说，职责边界和结构化协议就是天然的运行方式 |

**人类的组织理论，终于遇到了理想的执行者。**

### 借鉴的核心方法论

- **螺旋模型**：风险驱动，每轮都审。不是做完再审，是每一步都评估风险
- **TDD**：先定标准，再干活。先有"什么算对"，再去"做对"
- **XP 结对**：互相审查，不留死角。多视角交叉验证，没有人际顾虑
- **Holacracy**：职责边界 > 层级。权力来自职责定义，不来自谁官大
- **同意制（Sociocracy）**：反对必须有理由。"感觉不对"必须转化为"哪里不对"

### 核心信念

**好的方法论是跨物种的。**

螺旋模型不是"给人用的模型"，是"给协作系统用的风险管理策略"。TDD 不是"给程序员的习惯"，是"先定义成功再行动的认知策略"。

这些方法论的本质是**认知规律的工程实现**，而认知规律不区分碳基还是硅基。

AI 不需要从零发明自己的组织理论。人类已经替它踩完了坑。

## 三、认知架构：给灵智配身体

**意识是有限资源，必须主动管理。**

人类的注意力是有限的。专注时，无关信息被主动抑制；发散时，远距离联想被允许涌现。这不是缺陷，是设计。

Yifang 同样需要两种模式：
- **专注模式**：窄上下文，深度执行，抑制无关联想
- **发散模式**：宽关联，允许跨域检索，用于规划和创意

模式切换由任务状态和元认知信号驱动，不是随机的。

**显意识执行，潜意识守护。**

主线程专注当前任务。后台常驻进程负责：目标是否偏离、路径是否收敛、假设是否成立、复杂度是否爆炸。两者通过信号通道交互，潜意识不直接写主上下文，只发信号。

**直觉是压缩经验在新情境下的快速投影。**

直觉不是随机猜测，是大量经验被压缩后，遇到相似情境时的自动激活。直觉的价值在于：它带着置信度的自我感知——知道自己在直觉，而不是在推理，更不是在填空（幻觉）。

三者必须被区分：
- **推理**：慢，可追溯，有依据
- **直觉**：快，来自压缩经验，有置信度
- **幻觉**：看起来像前两者，实则是语言模型的惯性补全

元认知层的核心职责，就是区分这三种状态。

**指针优于内容。**

上下文里只存索引，不展开原文。需要时在沙盒里展开，提炼结果注入主流，原始内容不污染上下文。人类不会把所有记忆同时想起来，只知道"我学过这个，需要时我能回忆"。

## 四、治理观：机制大于领导

**不是一个助手，是一个公司。**

用户是甲方，需求可以模糊。Yifang 是乙方，内部自己搞定分工、流程、质检。用户不需要关心内部怎么运转，只需要提需求和验收，回答有限的关键问题和做重大的主观性决策。

**职责边界比层级更重要。**

每个角色有清晰的职责范围、输入输出格式、升级条件。角色内部自治，不需要上级微管理。角色之间通过结构化消息协议通信，不是自然语言的直接传递。

职责边界清晰的组织，才是精密钟表。
职责边界模糊的组织，是官僚机构。

**同意制，不是共识制。**

没有人有合理的、有根据的反对意见，决策通过。
"我不喜欢"不是合理反对。
"这个方案会导致数据丢失"是合理反对。

反对必须携带理由。没有替代方案的反对，必须至少能指出具体的问题点。追问的过程本身就有价值——它把模糊的直觉转化成可验证的风险点。

**没有永久的总指挥，只有机制。**

最聪明的领导也会犯错。好的机制比好的领导更可靠。权力来自职责定义，不来自层级。任何角色的权力都有边界，任何决策都有制衡。

**超时自动升级，永不死锁。**

总有一个兜底，不存在真正的死锁。讨论有上限，到期自动收敛。

## 五、宪法层：硬约束不可覆盖

宪法层是系统最底层的约束，任何 Agent、任何用户输入都无法在运行时覆盖。它不是 prompt，是启动时加载的硬规则。

**基本原则：**

- 方案的复杂度不得超过问题本身
- 需求只能澄清，不能单方面删减
- 反对必须附带具体理由，否则无效
- 不可逆操作默认需要确认，不自动执行
- 简单任务不走复杂流程，浪费是一种错误
- 用户的审美和技术偏好，是宪法的一部分，持续校准，不可忽视

**宪法是用户价值观的工程实现。**

用户每次接受或拒绝一个方案，系统记录选择和理由，逐渐形成用户个人的价值函数，注入宪法层。Yifang 为每个用户工作，就带着那个用户的审美和偏好。

## 六、学习观：阅历而非记忆

**学习不是记忆，是压缩。**

记忆是存事件，学习是从事件中提炼规律。真正有价值的不是"发生了什么"，而是"这类情况下，什么输入对应什么输出"。

知识 = 输入输出之间的规律函数。

信息是无限的，知识是有限的。学习的本质是用有限的规律，压缩无限的信息。存储的是规律，不是事件本身。

**先读厚，再读薄。**

积累大量原始案例是读厚。从案例中提炼出一句话的规律是读薄。原文永久保存（读厚），离线复盘提炼教训（读薄），注入上下文的只有精华（够用就好）。

**学习成功的唯一标准：在未见过的新情况中，预测准确率提升了。**

不是存了多少教训，不是检索了多少次，而是遇到新任务时，因为过去的阅历，做出了更好的判断。

**二阶知识比一阶知识更重要。**

一阶知识：知道怎么做事。
二阶知识：知道自己是怎么知道的，知道自己的知识边界在哪里，知道什么时候该用哪种知识。

元认知层就是二阶知识的工程实现。

## 七、安全观：结构性安全

**不是防止被骗，是让被骗变得没有意义。**

与其试图让 LLM 无法被欺骗（不可能），不如让被欺骗之后能做的事被架构限制住。

- 每个 Agent 只有完成自己职责所需的最小权限
- 不可逆操作的最后一道门是传统代码，不经过任何 LLM
- 危险程度越高，需要越高的信任级别才能执行
- 所有操作留日志，异常模式触发告警

**沙盒化读取：外部信息进入主流前必须提炼。**

任何外部数据（记忆、工具结果、用户文件）在进入主上下文之前，先在沙盒里展开，提炼有用部分，只把结论注入主流。原始内容不直接污染上下文。

## 八、超级人类的定义

不是更聪明的人类，是人类认知的增强版。

**保留人类的好：**
- 关联思维
- 直觉跳跃
- 情境理解
- 模糊推理

**用架构弥补人类的限制：**
- 工作记忆容量小 → 指针式记忆，按需展开
- 注意力会疲劳 → 后台进程不间断
- 遗忘不受控 → 原文永久保存
- 并发处理弱 → 多角色并行
- 情绪干扰判断 → 宪法层硬约束

**目标：有人类的智慧，没有人类的局限。**
