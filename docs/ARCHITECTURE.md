# Yifang 架构设计

> 本文档描述当前版本的具体实现。设计哲学见 [PHILOSOPHY.md](PHILOSOPHY.md)。

---

## 核心机制：动态异步书面会议

每次用户输入触发一场会议。会议内部角色各司其职，通过结构化消息（JSON）协作，书面异步，每个字都有记录。

### 角色体系

**常驻角色（不可撤销）：**

| 角色 | 职责 | 独立性 |
|------|------|--------|
| **秘书** | 信息守门人。摄入压缩、轮后压缩、最终输出整理 | 服务于会议流程 |
| **主控** | 只做决策，不做内容。决定召集/解散哪些专家，决定继续还是收敛 | 受宪法约束 |
| **安全审计** | 独立监控全程，三级风险响应（LOW/MEDIUM/HIGH） | 主控无法解散 |
| **质疑者** | 对专家发言进行质疑，找出逻辑漏洞和盲区 | 主控无法解散 |

**按需角色（主控动态调度）：**

| 角色 | 职责 |
|------|------|
| **常识推理专家** | 识别核心对象、前提条件、逻辑关系 |
| **伦理顾问** | 识别伦理争议、身份立场、敏感点 |
| **领域通才** | 补充遗漏角度，跨域知识 |
| **风险官** | 识别关键风险，区分重要性 |

专家按需召集、完成即离场，不固定参与全程。

### 会议流程

```
用户输入
  ↓
秘书摄入（短内容透传，长内容压缩）
  ↓
安全审计（审计原始输入 + 摘要）
  ├─ HIGH → 暂停会议
  └─ 继续
  ↓
主控首轮决策（召集哪些专家）
  ↓
┌─ 最多 5 轮 ──────────────────────────┐
│ 专家并行发言                          │
│   ↓                                   │
│ 质疑者审视                            │
│   ↓                                   │
│ 安全审计（审计本轮内容）              │
│   ↓                                   │
│ 主控决策（继续/收敛/等待用户）        │
│   ↓                                   │
│ 秘书压缩本轮（注入下轮上下文）        │
└───────────────────────────────────────┘
  ↓
秘书编译结论 → 整理为自然语言回答
  ↓
用户获得结果
```

### 信息流设计

- **秘书压缩**：长内容（>1000 字符）自动由 LLM 压缩为摘要；短内容直接透传
- **轮后压缩**：每轮结束后，本轮专家发言 + 质疑 + 主控决策被压缩为下轮上下文
- **主控隔离**：主控始终只接收摘要，不接触原文

### 权力制衡

- 安全审计不可撤销，永驻监督
- 质疑者不可撤销，独立质证
- 主控有调度权，但受宪法约束（必须响应中高风险警告）
- HIGH 级风险绕过主控，直接暂停——这是代码层硬约束

### 宪法约束（当前实现）

宪法约束硬编码在代码和 prompt 中：
- 主控无法解散安全审计和质疑者
- 主控必须响应中高风险警告
- HIGH 级风险直接暂停会议

### 可观测性

- **TokenTracker**：记录每个角色的调用次数和 token 消耗，按角色明细展示
- **RoundRecord**：完整记录每轮的决策、发言、审计结果、压缩摘要
- **调试模式**：可输出完整会议记录

## 哲学映射

| 设计哲学 | 架构实现 |
|---------|---------|
| 专注 vs 发散 | 主控聚焦决策（专注），多专家并行分析（发散） |
| 显意识 vs 潜意识 | 主控推进主流程，审计和质疑者后台守护 |
| 指针优于内容 | 秘书压缩，主控只接收摘要 |
| 螺旋模型 | 每轮都经过质疑→审计→决策的风险评估 |
| TDD | 任务约束先于执行，质疑基于约束而非事后评判 |
| XP 结对 | 多专家交叉审视 + 质疑者独立质证 |
| Holacracy | 职责定义权力，主控不能解散审计 |
| 同意制 | 质疑者反对必须指出具体问题 |
| 超时兜底 | 最多 5 轮，自动收敛 |

## 远期愿景

以下概念是设计方向，尚未在当前版本中实现。

- **Circle 结构**：将不同类型的任务（编码、写作、分析）拆分为独立 Circle，每个 Circle 内部运行自己的会议，Circle 之间通过结构化协议通信
- **任务路由**：简单任务接待层直接解决；中等任务转给对应 Circle；复杂任务走完整流程
- **Stigmergy（痕迹协调）**：去中心化协作，Agent 完成任务后在共享环境留下痕迹，其他 Agent 读取痕迹自动知道下一步
- **信用系统**：异议的历史准确率决定未来权重。多次直觉正确的角色，其信号在未来得到更高权重
- **分层决策权**：局部影响→自主决定；跨域影响→同意制；宪法级→极高门槛
- **阅历系统**：三层存储（原文/教训/偏好），教训有生命周期（时间衰减、验证强化、主动失效），注入上下文 O(1) 固定上限
- **宪法参数化**：用户每次接受或拒绝方案，系统记录选择和理由，逐渐形成用户个人的价值函数
